  <action>
    Create `connector/docs/INSTALLATION.md` covering:

    **1. System Requirements:**
    - Windows 10, Windows 11, Windows Server 2016/2019/2022
    - x64 architecture
    - .NET runtime NOT required (bundled in MSI)
    - Network: outbound TCP port 443 only (no inbound ports needed)
    - SQL Server: network access to the Primavera P6 database (SQL Server or Windows auth)

    **2. Standard Installation:**
    - Double-click MSI, follow wizard
    - Service installs as "Apura Connector" under LocalService account
    - Auto-starts on boot
    - Default install path: `C:\Program Files\Apura\Apura Connector\`

    **3. Configuration:**
    - Edit `appsettings.json` in install directory for non-sensitive settings (TunnelEndpoint, MaxConcurrentQueries, etc.)
    - For SQL Server credentials: run the credential setup tool or configure via DPAPI (explain that credentials are encrypted at rest)
    - API key configuration
    - Restart service after config changes: `Restart-Service ApuraConnector` (PowerShell) or via services.msc

    **4. Silent/GPO Deployment:**
    - `msiexec /i ApuraConnector.msi /quiet /norestart /l*v install.log`
    - Uninstall: `msiexec /x ApuraConnector.msi /quiet /norestart`
    - Upgrade: same `/i` command with new MSI (MajorUpgrade handles it)
    - GPO: assign MSI via Group Policy Software Installation

    **5. Network/Firewall Requirements:**
    - Only outbound TCP 443 to `*.apura.xyz` required
    - The MSI automatically creates a Windows Firewall outbound rule
    - No inbound ports needed (connector initiates all connections)
    - Proxy: if outbound HTTPS goes through a proxy, configure in appsettings.json
    - For restrictive firewalls: whitelist `ws.apura.xyz` and `api.apura.xyz`

    **6. Auto-Update:**
    - Connector checks for updates every 6 hours
    - Updates download automatically and install via Windows Task Scheduler
    - The service restarts automatically after upgrade
    - To disable: set `UpdateCheckIntervalHours: 0` in appsettings.json

    **7. Troubleshooting:**
    - Logs location: `C:\Program Files\Apura\Apura Connector\logs\`
    - Service status: `Get-Service ApuraConnector` (PowerShell)
    - Common issues: SQL Server connectivity, API key invalid, firewall blocking outbound
    - Event log: check Windows Event Log > Application for ApuraConnector entries

    **8. Uninstallation:**
    - Via Add/Remove Programs or `msiexec /x` command
    - Removes service, firewall rules, and program files
    - Credential file (credentials.dpapi) is preserved for reinstallation

    Keep it practical and concise. Target audience is enterprise IT admins who deploy via GPO/SCCM. No marketing language.
