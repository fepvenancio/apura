## Configuration

**Environment:**
- `.env.example` present in `frontend/` - Documents required env vars
- Key env var: `NEXT_PUBLIC_API_URL` - Frontend API endpoint (defaults to `http://localhost:8787`)
- Cloudflare env vars: `CLAUDE_API_KEY`, `STRIPE_WEBHOOK_SECRET` (optional), `JWT_SECRET`, `INTERNAL_SECRET`

**Build:**
- `frontend/tsconfig.json` - TypeScript strict mode, ES2017 target, path alias `@/*` → `./src/*`
- `frontend/next.config.ts` - Static export mode (`output: "export"`), unoptimized images
- `frontend/postcss.config.mjs` - Tailwind CSS 4 PostCSS plugin
- Root `package.json` - Turbo monorepo with workspaces: `packages/*` and `frontend`

## Platform Requirements

**Development:**
- Node.js >= 20.0.0
- npm 10.9.0+
- wrangler CLI for local Cloudflare development
- .NET 8.0 SDK (for connector builds)
- TypeScript 5.7.0+

**Production:**
- Cloudflare Workers for backend (api-gateway, ai-orchestrator, ws-gateway)
- Cloudflare D1 for database storage
- Cloudflare KV for caching/sessions
- Cloudflare Durable Objects for WebSocket state (ws-gateway)
- Statically hosted frontend (export Next.js output to CDN/Pages)
- .NET 8.0 runtime for on-prem Primavera connector service

## Build & Deployment

**Monorepo Tool:**
- Turbo 2.4.0 - Task orchestration for build, dev, lint, typecheck, deploy across all workspaces

**Scripts (Root):**
- `npm run dev` - Start all packages in development mode via Turbo
- `npm run build` - Build all packages
- `npm run test` - Run tests across all packages
- `npm run lint` - Lint all packages
- `npm run typecheck` - Type-check all packages
- `npm run deploy` - Deploy all packages to Cloudflare/hosting
- `npm run db:migrate` - Run database migrations via `deploy/migrate.sh`

**Wrangler Config Files:**
- `packages/api-gateway/wrangler.toml` - D1 database, KV cache, service bindings to AI_ORCHESTRATOR and WS_GATEWAY
- `packages/ai-orchestrator/wrangler.toml` - D1 database, KV cache, Claude model config
- `packages/ws-gateway/wrangler.toml` - D1 database, KV cache, Durable Objects for ConnectorSession
- Custom domain routing: `api.apura.xyz`, `ws.apura.xyz`
